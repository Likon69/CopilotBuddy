[Guide] How to write a basic Questing Profile

Par Maffyx — publié le 26 août 2015 sur RebornBuddy (rebornbuddy.com)
Source : https://rebornbuddy.com/xf/threads/guide-how-to-write-a-basic-questing-profile.223620/

Hello All, I've seen a lot of requests around the forums for a guide on how to write profiles. I've seen a lot of requests for grinding profiles but I feel that using the questing bot is more useful since we can take advantage of Quest Behaviors and this can make our profiles more robust. This guide is not meant to be all inclusive but to be a starting point for those who want to begin building profiles.

1. Text Editor Setup

I personally use Notepad++ to write my profiles. It allows me to quickly edit and change text without much hassle.

  • Download: https://notepad-plus-plus.org/
  • Recommended theme: Settings > Style Configurator > Select Theme > Obsidian
  • For XML color: go to Language column > XML > change comment foreground to lime green so comments stand out

2. Setting Up an SVN (for Distribution)

If you want to distribute your profiles, set up an SVN. Recommended tools:

  • Assembla — free SVN hosting
  • TortoiseSVN — Windows SVN client
  1. Create your SVN account on Assembla; get your SVN trunk link from SVN tab > Instructions sub tab
  2. Create a folder (not inside the Honorbuddy folder), e.g. on your Desktop
  3. Right-click the folder > SVN Checkout > paste your trunk URL > OK
  4. Recommended folder structure: 
    Farming/
  Classic/
  Burning Crusade/
  Wrath of the Lich King/
  Cataclysm/
  Mists of Pandaria/
  Warlords of Draenor/
Reputation/
  ...same sub-folders...
  5. Save your profiles to the trunk folder

3. Resources

a. Honorbuddy Developer Tools

The most important tool for building profiles — built into the HB interface.

b. Quest Behaviors

Found in your Honorbuddy folder under Quest Behaviors/. Main behaviors:

  • InteractWith — Interacts with a mob or object; many variables
  • KillUntilComplete — Grinds mobs until a condition is met
  • UseItem — Uses an item in your bag
  • RunMacro — Runs an in-game macro command

c. Documentation Links

  • Profile Tags: HBProfile Grammar

  • Condition Examples: Condition examples
  • API Docs: docs.honorbuddy.com

4. Test Profile Structure

A basic profile skeleton:

<HBProfile>
  <QuestOrder>

    <!-- The Crone's Bargain -->
    <While Condition="!HasQuest(12841) && !IsQuestCompleted(12841)">
      <PickUp Nav="Fly" QuestName="The Crone's Bargain" QuestId="12841"
              GiverName="Lok'lira the Crone" GiverId="29481" />
    </While>

    <While Condition="HasQuest(12841) && !IsQuestCompleted(12841)">
      <CustomBehavior File="InteractWith" QuestId="12841" MobId="29518"
              CollectionDistance="200" X="6929.614" Y="-1316.467" Z="831.2584" />
    </While>

    <While Condition="HasQuest(12841) && IsQuestCompleted(12841)">
      <TurnIn Nav="Fly" QuestName="The Crone's Bargain" QuestId="12841"
              TurnInName="Lok'lira the Crone" TurnInId="29481" />
    </While>
    <!-- End The Crone's Bargain -->

  </QuestOrder>
</HBProfile>

While Loop Logic (the 3-loop pattern)

  1. !HasQuest(X) && !IsQuestCompleted(X) → Pick up the quest
  2. HasQuest(X) && !IsQuestCompleted(X) → Complete the quest objective(s)
  3. HasQuest(X) && IsQuestCompleted(X) → Turn it in
Note XML: < and > must be escaped in condition attributes: &lt; for <, &gt; for >, &amp;&amp; for &&.
Reference: XML character entities (Wikipedia)

5. Using Quest Overrides

Overrides are placed outside the <QuestOrder>, usually above it. They refine how the bot handles a specific quest (e.g. better hotspots).

<Quest Id="28145" Name="Venomblood Antidote">
  <Objective Type="KillMob" MobId="47748" KillCount="8">
    <Hotspots>
      <!-- Add hotspot coordinates here -->
    </Hotspots>
  </Objective>
</Quest>

<QuestOrder>
  ...
</QuestOrder>

6. XML Profile Tags — Developer Tools Quick Reference

PickUp / TurnIn (from Dev Tools copy XML)

<PickUp Nav="Fly" QuestName="Venomblood Antidote" QuestId="28145"
        GiverName="Sun Priest Asaris" GiverId="47715" />
<Objective QuestName="Venomblood Antidote" QuestId="28145"
        Type="KillMob" Nav="Fly" MobId="47748" KillCount="8" />
<TurnIn Nav="Fly" QuestName="Venomblood Antidote" QuestId="28145"
        TurnInName="Sun Priest Asaris" TurnInId="47715" />

Vendor / NPC (from Dev Tools — Current Target)

<Vendor Name="Innkeeper Allison" Entry="6740" Type="Food"
        Nav="Fly" X="-8867.786" Y="673.6729" Z="97.90324" />
  • Entry = Unit ID (also visible on Wowhead: wowhead.com/npc=6740)
  • Type: Food, Repair, Innkeeper — only matters for vendor sections
  • XYZ — coordinates in the game world

Blackspot

<Blackspots>
  <Blackspot X="-5137.301" Y="544.6486" Z="84.33643" Radius="22.15761" />
  <Blackspot X="-5117.417" Y="537.1185" Z="85.25964" Radius="22.15761" />
</Blackspots>
Important (thebrodieman): Blackspots don't just punch a hole in the mesh — any mesh segment that intersects the blackspot radius becomes entirely blackspotted. Keep them small.

Hotspot

<Hotspot X="-1441.929" Y="9866.271" Z="200.9235" />

Generated via Dev Tools > Generate Hotspot button (your character's current position).

7. Developer Tools Tabs Reference

Profiles Tab

  • Local Player Info — current position (hit Refresh to update)
  • Current Target Info — Copy XML button: generates PickUp/TurnIn/Vendor XML for whatever you have targeted
  • Common Buttons — Generate Blackspot (walk edge-to-edge), Generate Hotspot (current pos)
  • Quests — shows quest log; right-click > Copy XML to auto-generate PickUp + Objective + TurnIn

Console Tab

  • Test bed for scripts/C# expressions before putting them in profiles or plugins
  • New — fresh window; Delete — remove script; Run (F5) — debug/run
  • Click to set Keybind — bind a common test script to a key

Objects Tab — Sub-tabs

GameObjects: Mailboxes, chairs, interactable objects. Key columns: Entry (MobId for InteractWith), Name, Position (XYZ), Distance, Type, SubType.

Units: All non-player NPCs + player pets. Key columns: Entry (MobId/UnitId), Name, Position, FactionID, Distance, NPC Flags.

Bag Items: Everything in bags, reagent bank, personal bank. Key columns: Entry (ItemId for UseItem), Name, Stack Count.

Auras Tab

  • Auras on Me — SpellID, Name, StackCount, AuraFlags, TimeLeft, Creator, CreatorGUID
  • Auras on Current Target — same info for targeted unit

SpellID useful for HasAura() conditions and InteractWith requirements.

Merchant Items Tab

Open a vendor window in-game first. Shows: ItemId, Name, Quantity, BuyPrice (copper), NumAvailable (-1 = infinite).

Targeting Tab

When the bot is running and killing/collecting, shows active kill targets and lootable mobs within loot radius.

8. Publishing Your Profile

  1. Make sure the file is in the correct SVN trunk folder (don't move it later)
  2. Right-click the file > SVN Commit → enter credentials if prompted
  3. Navigate to your SVN trunk URL, right-click the file > Copy Link Address
  4. Post that link in your forum thread for distribution

10. Developer Tools — Vue détaillée

Profiles Tab (détail)

Quand tu cibles un NPC et cliques Copy XML sur Current Target Info, tu obtiens :

<Vendor Name="Forest Spider" Entry="30" Type="Repair"
        Nav="Fly" X="-9226.476" Y="272.2993" Z="72.97188" />

L'Entry est synonyme de UnitId, MobId, NpcId — c'est la même valeur que sur Wowhead (wowhead.com/npc=30).

Pour les quêtes, Copy All Quests génère le XML pour toutes les quêtes du journal. Copy Overrides génère les blocs <Quest Id="..."> pour chacune.

Refresh : toujours appuyer sur Refresh après avoir bougé ou changé de cible avant de copier les coordonnées.
